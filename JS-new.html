WHAT IS THE Execution Context?

An execution context is like a big box where JavaScript code is evaluated and executed. It has two main components:

Memory Component (also called the Variable Environment)
This is where all variables and function declarations are stored in memory during the initial phase.

Code Component (also called the Thread of Execution)
This is where the code is executed line by line, values are assigned, and functions are invoked.

JavaScript can only execute one command at a time. It is a synchronous, single-threaded language, which means:

Code runs one line at a time, in order. It canâ€™t move to the next line until the current line is completely executed.
This is what we mean when we say JavaScript is a synchronous single-threaded language.

apply() - The apply() method in JavaScript is a function method that allows a function to be called with a specified this value and arguments provided as an array or an array-like object.

Event Propagation

Event propagation in JavaScript describes the mechanism by which events travel through the Document Object Model (DOM) tree when an event occurs on an element.

Events in the DOM tree go through 3 phases:
1. Capturing phase (top to target)
2. Target phase
3. Bubbling phase (target back to top)
You can control propagation using:
event.stopPropagation() â€” Stops the event from bubbling up or down
event.preventDefault() â€” Prevents default behavior (like form submission)

Capturing Phase: The click event would first be captured by window, then document, then html, then body, then the div, and finally the button.

Bubbling Phase: The click event would then bubble up from the button to the div, then body, then html, then document, and finally window.

Target Phase: The event reaches the actual element on which it was triggered.

function findDuplicates(nums) {
  const count = {};  // To store how many times each number appears 
  const result = [];  // To store duplicates 

  for (let num of nums) {

      function generatePrimesUpTo(limit) {
  const sieve = new Array(limit + 1).fill(true);
  sieve[0] = sieve[1] = false;
  
  for (let i = 2; i <= Math.sqrt(limit); i++) {
    if (sieve[i]) {
      for (let j = i * i; j <= limit; j += i) {
        sieve[j] = false;
      }
    }
  }
  
  return sieve
    .map((isPrime, num) => isPrime ? num : null)
    .filter(num => num !== null);
}

import { EmployeeService } from './employee.service';
import { from } from 'rxjs';
import { map, toArray, distinct, switchMap, filter } from 'rxjs/operators';

@Component({
  selector: 'app-employee',
  templateUrl: './employee.component.html',
})
export class EmployeeComponent implements OnInit {
  secondHighestSalary: number | null = null;

  constructor(private employeeService: EmployeeService) {}

  ngOnInit() {
    this.employeeService.getEmployees().pipe(
      switchMap(employees => from(employees.map(e => e.salary))),  // flatten to salary stream
      distinct(), // remove duplicates
      toArray(), // collect into array again
      map(salaries => salaries.sort((a, b) => b - a)), // sort descending
      map(sorted => sorted.length > 1 ? sorted[1] : null) // get second highest
    ).subscribe(second => {
      this.secondHighestSalary = second;
      console.log('Second highest salary:', second);
    });
  }
}

function validateForm() {
  const name = document.getElementById("name").value.trim();
  const email = document.getElementById("email").value.trim();
  const phone = document.getElementById("phone").value.trim();
  const errorMsg = document.getElementById("errorMsg");

  // Name validation
  if (name === "") {
    errorMsg.textContent = "Name is required.";
    return false;
  }

  // Email validation
  const emailPattern = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  if (!emailPattern.test(email)) {
    errorMsg.textContent = "Please enter a valid email.";
    return false;
  }

  // Phone validation (10 digits)
  const phonePattern = /^[0-9]{10}$/;
  if (!phonePattern.test(phone)) {
    errorMsg.textContent = "Phone must be 10 digits.";
    return false;
  }

  errorMsg.textContent = ""; // Clear error if everything is valid
  alert("Form submitted successfully!");
  return true;
}

Responsive Web Design (RWD) ?

Responsive Web Design is an approach to web design that makes web pages render well on a variety of devices and window or screen sizes. It uses flexible layouts, media queries, and responsive images to adapt the layout to the viewing environment.

Key components of Responsive Design?

  • Flexible grid layouts
  • Flexible images and media
  • Media queries
  • Viewport meta tag

Media query

Media queries are a CSS technique used to apply styles based on device characteristics like screen width, height, resolution, and orientation.

Viewport meta tag

It controls how a webpage is displayed on mobile devices.

Mobile-First approach

Designing the mobile layout first, then scaling up to larger screens using media queries (min-width). It promotes performance and usability on smaller devices.

Flexbox and Grid help in Responsive Layouts

Docker & Kubernetes

What is Docker?

Docker is a containerization platform that allows developers to package applications and their dependencies into lightweight, portable containers.

What is Kubernetes?

Kubernetes is an open-source container orchestration platform used to automate the deployment, scaling, and management of containerized applications.

How is Docker different from a Virtual Machine?

Docker containers share the host OS kernel and are more lightweight. VMs run full OS instances and are more resource-intensive.

What are the main components of Docker?

  • Docker Engine
  • Docker Images
  • Docker Containers
  • Docker Hub
  • Docker Compose

Docker Images and Containers?

Images are read-only templates used to create containers.
Containers are the running instances of these images.

Dockerfile?

A Dockerfile is a script containing instructions to build a Docker image.

Pod in Kubernetes?

A Pod is the smallest deployable unit in Kubernetes. It can contain one or more containers sharing network and storage.

What is the role of the Kubelet in Kubernetes?

Kubelet is an agent that runs on each node and ensures containers in a Pod are running as expected.

What is a ReplicaSet in Kubernetes?

It ensures a specified number of pod replicas are running at all times.

Deployment in Kubernetes?

A Deployment manages ReplicaSets and Pods, allowing you to declare and manage updates for stateless applications.

How does Kubernetes manage service discovery?

Kubernetes uses Services, DNS, and labels to enable Pods to discover and communicate with each other.

What is the difference between a Service and Ingress in Kubernetes?

  • Service: Exposes a set of Pods (internally or externally)
  • Ingress: Manages HTTP/S access and routes traffic to the correct service

StatefulSet

StatefulSet is used for managing stateful applications that require persistent identity and storage.

Helm

Helm is a Kubernetes package manager that uses Helm Charts to define, install, and upgrade complex Kubernetes applications.

ConfigMaps and Secrets

  • ConfigMaps store non-sensitive configuration data.
